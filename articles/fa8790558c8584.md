## Agent Skillsに対する僕の理解
Agent Skillsについての一般的な説明は散々他の記事や公式ドキュメントでなされているので、僕なりの理解を示そうと思います。

### Agent Skillsとはプロンプトの遅延評価である
ドキュメントの中では段階的開示（progressive disclosure）という用語が使われていますが、個人的にはプロンプトの遅延評価の方がしっくりきています。
その名の通り、Agent Skillsはプロンプトを遅延評価するためのフォーマットです。必要なときに必要な知識を注入することができます。
現在のエージェンティックコーディングでは、コンテキスト管理が重要になっています。コンテキストが大きくなりすぎると、重要な決定を忘れやすくなってしまい作業効率が落ちてしまいます。CLAUDE.mdにプロンプトを書くと、claudeを立ち上げたときに常に読み込まれるため、セッション中に時々しか必要のない知識が余計にコンテキストを圧迫してしまいます。
Agent Skillsは、必要なときに必要な知識を注入するため、コンテキスト管理がしやすくなります。

### Agent Skillsとは専門知識を詰め込むためのフォーマットである
Agent Skillsにはスキル本体（SKILL.md）以外に、実行可能なスクリプトや追加のマークダウンを含めて置くことができます。このアーキテクチャは特に強力だと思っていて、Slash Commandsの頃は、コマンド本体しか配置できず、その他のリソースの専用の置き場所はないので散らかっていました。Agent Skillsは、ディレクトリとして管理され、その中に必要なリソースをまとめて置くことができるため、専門知識を詰め込むのに適しています。
リソースに対しても遅延評価が働くため、適切にリソースを配置することでコンテキストウィンドウの消費を抑えることができます。

## 良い感じのAgent Skillsを作るために意識していること
次にいくつかスキルを作ったり改善していく中で、良い感じのスキルを作るために意識していることを紹介します。

### 自分の仕事について振り返る
何をスキルにしますか？と聞かれたときに、まずは自分の仕事について振り返ることが大事だと思います。自分の仕事の中で、どんな作業があって、どんな知識が必要で、どんな判断をしているのかを振り返ることで、スキルにするべき作業や知識が見えてくると思います。

例えば、質の高いユニットテストを書くために、僕が意識していることは以下のとおりです。
1. テスト対象のコンポーネントがどのような振る舞いを実装しているのかを理解する
  1. 外部から観測可能な振る舞いと実装の詳細を切り分けて考える
2. どのようなテストケースが必要なのかを考える
  1. テストされていない振る舞いがないかを考える
  2. AAAパターンを使うとか、モックよりもフェイクを使うとか、そういったユニットテストに関する具体的なプラクティスもあります
3. テストケースを実装する
これらの知識をもとに、スキルにするべき作業や知識を考えると、例えば以下のようなものが考えられます。
- analyze-component-behavior: テスト対象のコンポーネントの振る舞いを分析する作業
- plan-test-cases: テストケースを計画する作業
- unit-test-guidelines: ユニットテストに関するガイドライン

### 決定論的作業と非決定論的作業の切り分け
作業の中には、決定論的な作業と非決定論的な作業があります。決定論的な作業は、スクリプト化可能な作業です。例えば、現在のブランチ名を取得する作業は、決定論的な作業です。一方、非決定論的な作業は、スクリプト化が難しい作業です。例えば、ブランチ名を考える作業は、非決定論的な作業です。
非決定論的作業はプロンプトとして記述して、エージェントに推論させるのが良いと思います。決定論的作業はスクリプトとして記述して、エージェントに考えさせないことでコンテキストウィンドウの無駄使いを防ぐのと、アウトプットの安定性を高めることができます。

### 改善を繰り返す
スキルは一度作ったら終わりではなく、改善を繰り返すことが大事です。スキルを使ってみて、どんなときにうまくいって、どんなときにうまくいかないのかを観察することで、スキルの改善点が見えてきます。
うまく呼び出されない場合はdescriptionを改善して、自分の実際の作業から推測しやすいものにするのが良いでしょう。
作ってみたけど、役に立っていないと思ったらそのスキルは削除しましょう。スキルの名前とdescription自体は毎回コンテキストウィンドウに入るので、役に立っていないスキルを残しておくのはコンテキストウィンドウの無駄使いになってしまいます。

## ぶっちゃけSkillちゃんと使えてる？
2026年1月中旬くらいまでは、Agent Skillsはドキュメントに書いてあるほどよしなに使ってくれなかったので、これならコマンドで十分だなと思っていました。しかし、最近のclaudeはよしな力が上がっていてスキルが発動する確率が上がっているような気がしています。それとコマンドとスキルが統合されたことにより、コマンドとして作るよりもスキルとして作っておいた方がわんちゃんよしなに呼んでくれる時もあるかなと思って基本はスキルとして作るようにしています。
